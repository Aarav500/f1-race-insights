# F1 Race Prediction: Technical Specification

## 1. Problem Formulation

### 1.1 Objective

Predict Formula 1 race outcomes given pre-race information:
- **Win probability**: P(driver finishes 1st | pre-race features)
- **Podium probability**: P(driver finishes ≤ 3rd | pre-race features)
- **Expected finish**: E[finish position | pre-race features]

### 1.2 Challenges

- **Temporal dynamics**: Driver/team performance evolves
- **Limited data**: ~20-24 races/season, ~20 drivers/race
- **Data leakage risk**: Must use only strictly prior information
- **Imbalanced outcomes**: 1 winner, 3 podiums per race
- **Interdependencies**: Driver outcomes not independent

---

## 2. Data

### 2.1 Source

FastF1 API providing official F1 timing data (2020-2025).

### 2.2 Raw Data

- Race results: finish position, status, points
- Qualifying results: grid positions, lap times
- Metadata: season, round, track, teams

### 2.3 Feature Engineering

**Temporal rolling features** (strict no-leakage):

**Qualifying**:
- `quali_position`: Grid position (1-20)
- `quali_delta_to_pole`: Time to pole
- `quali_delta_to_teammate`: vs teammate

**Driver form (rolling)**:
- `driver_rolling_avg_finish`: Avg finish (lower better)
- `driver_rolling_avg_points`: Avg points
- `driver_rolling_dnf_rate`: DNF frequency

**Constructor form (rolling)**:
- `constructor_rolling_avg_finish`
- `constructor_rolling_avg_points`
- `constructor_rolling_dnf_rate`

**Data leakage prevention**: Rolling features use **strictly prior races only**.

---

## 3. Baseline Models

### 3.1 Qualifying Frequency

Empirical probabilities from historical data:

```
P(win | quali_pos = k) = #{wins from k} / #{starts from k}
```

**Advantages**: Simple, interpretable  
**Limitations**: Ignores driver/team identity

### 3.2 Elo Rating

Pairwise rating system updated chronologically:

1. Initialize: R₀ = 1500
2. Update after race: R'ᵢ = Rᵢ + K × (Sᵢⱼ - E(Rᵢ, Rⱼ))
3. Predict: softmax over ratings

**Advantages**: Captures driver quality, temporal evolution  
**Limitations**: Doesn't use detailed features

## 4. Model Zoo

Supervised learning models trained on (features, outcomes).

### 4.1 Logistic Regression

Linear model: P(win) = σ(w^T x + b)

### 4.2 Random Forest

Ensemble of decision trees with bootstrap aggregation.

### 4.3 Gradient Boosting

XGBoost, LightGBM, CatBoost - iteratively add weak learners.

**Parameters**:
- Trees: 100
- Max depth: 6
- Learning rate: 0.1
- Loss: LogLoss (classification), MSE (regression)

## 5. NBT-TLF: Neural Bradley-Terry with Temporal Latent Factors

### 5.1 Motivation

Pairwise ranking formulation:
- Learn from relative comparisons
- Embed drivers/constructors/tracks
- Model temporal evolution

### 5.2 Architecture

**Embeddings**:
- Driver: dᵢ ∈ ℝ^d
- Constructor: cᵢ ∈ ℝ^d
- Track: tₖ ∈ ℝ^d

**Temporal encoding** (sinusoidal):
- race_idx → τ(race_idx) ∈ ℝ^t

**Score function**:
```
s(driver, constructor, track, time) = MLP([d; c; t; τ(time)])
```

MLP: [concat_dim → hidden → hidden/2 → 1]

**Pairwise prediction** (Bradley-Terry):
```
P(i beats j) = σ(sᵢ - sⱼ)
```

### 5.3 Training

- Data: Pairwise comparisons from race results
- Loss: Binary cross-entropy
- Optimizer: Adam (lr=0.001, weight_decay=0.01)
- Epochs: 50 with early stopping

### 5.4 Inference

For race prediction:
1. Compute score sᵢ for each driver
2. Softmax: P(win | i) = exp(sᵢ) / Σⱼ exp(sⱼ)

---

## 6. Evaluation Protocol

### 6.1 Walk-Forward Backtesting

Temporal cross-validation (expanding window):

1. Sort races chronologically
2. Start with N races as training
3. For each test race:
   - Train on all prior races
   - Predict on current race
   - Add to training set

**No look-ahead bias**: only prior data used.

### 6.2 Metrics

**Classification** (win/podium):
- **AUC**: Discrimination (0.5=random, 1=perfect)
- **LogLoss**: Proper scoring rule (lower better)
- **Brier**: (p - y)² (calibration + discrimination)
- **ECE**: Expected Calibration Error (binned gaps)

**Regression** (finish):
- **MAE**: Mean absolute error

---

## 7. Calibration

### 7.1 Motivation

Raw model outputs may not be well-calibrated probabilities.

### 7.2 Methods

**Tree models**: Isotonic regression on validation set

**NBT-TLF**:
1. Softmax within race (sum-to-one)
2. Optional isotonic calibration
3. Normalization

### 7.3 Validation

- Probabilities in [0, 1]
- Win probs sum to 1 per race
- Win prob ≤ podium prob

---

## 8. Interpretability

### 8.1 Tree Models (XGB/LGBM/Cat)

**Method**: SHAP (Shapley Additive exPlanations)
- TreeExplainer for efficiency
- Exact Shapley values
- Feature contributions

### 8.2 Linear/Ensemble (LR/RF)

**Method**: Permutation Importance
- Permute each feature
- Measure performance drop
- Global importance

### 8.3 NBT-TLF

**Method**: Component Ablation
- Zero-out embeddings
- Measure score delta
- Contributions:
  - Driver embedding
  - Constructor embedding
  - Track embedding

## 9. Counterfactual Analysis

### 9.1 Purpose

Answer "what-if" questions:
- "What if Hamilton qualified P1 instead of P3?"
- "What if Red Bull's reliability improved?"

### 9.2 Method

1. Baseline: Predict with original features
2. Apply delta: Modify feature(s)
3. Counterfactual: Re-predict
4. Compare: Report delta

**Supported deltas**:
- `qualifying_position_delta`: Grid position change
- `driver_form_delta`: Driver performance adjustment
- `constructor_form_delta`: Team performance
- `reliability_risk_delta`: DNF rate change

### 9.3 Delta Semantics

- Qualifying: Negative = better (e.g., -2 means P3→P1)
- Form: Positive = better (multiplicative, 0.1 = 10% improvement)
- Reliability: Additive (±0.05 = ±5% DNF risk)

**Clamping**: Qualifying ∈ [1, 20], rates ∈ [0, 1]

## 10. Limitations

### 10.1 Data Limitations

- Limited history: 5-6 seasons of modern data
- Sparse events: 20-24 races/season
- Driver/team changes complicate temporal modeling

### 10.2 Modeling Assumptions

- **Independence**: Treats driver outcomes as independent (they're not)
- **Feature completeness**: Missing strategic factors (tire choice, weather)
- **Linear effects**: Deltas assume linear/multiplicative effects

### 10.3 Evaluation Constraints

- No future validation until data exists
- New circuits have no historical data

### 10.4 Future Work

- **Race simulation**: In-race dynamics (overtaking, pit stops)
- **Multi-task learning**: Joint qualifying + race prediction
- **Attention mechanisms**: Model driver-driver interactions
- **External data**: Weather, tire compounds, regulations
- **Uncertainty quantification**: Bayesian approaches
